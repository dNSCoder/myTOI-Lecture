Chapter 01

Tree คืออะไร?

⏱ ~20 min

Tree คือโครงสร้างข้อมูลแบบ Hierarchical — ข้อมูลจัดเป็นลำดับชั้นจากบนลงล่าง ต่างจาก Array ที่เรียงตรงๆ หรือ Graph ทั่วไปที่ไม่มีทิศทาง

ใน CP เรามอง Tree ว่าเป็น Graph พิเศษ ที่มีสมบัติครบ 3 ข้อ:

⚡ CP Definition

1. Connected — ทุก node ถึงกันได้
2. Acyclic — ไม่มี cycle
3. n nodes มี n−1 edges เสมอ — มากกว่านี้คือ cycle, น้อยกว่านี้คือ disconnected

โจทย์ CP มักให้ input เป็น n และ n-1 edges แล้วให้เดิน DFS/BFS — ไม่ได้สร้าง struct Node ให้เลย

Tree คือ Graph ประเภทหนึ่ง — แต่มีข้อจำกัดพิเศษที่ทำให้ code ง่ายขึ้นมาก:

ลักษณะGraph ทั่วไปTree
Cycleมีได้❌ ไม่มีเลย
การเชื่อมต่ออาจ disconnected✅ ทุก node ถึงกันเสมอ
จำนวน edges0 ถึง n(n-1)/2✅ n−1 edges เสมอ
Path ระหว่าง 2 nodeอาจมีหลายเส้น✅ มีเส้นเดียวเสมอ
ต้องใช้ visited array?✅ ต้องใช้❌ ส่ง parent มาแทนได้
💡 ผลที่ได้จากข้อจำกัดของ Tree

เพราะไม่มี cycle และมี unique path — DFS บน Tree ทำได้ง่ายกว่า Graph มาก
แค่ส่ง parent มาป้องกันการเดินย้อน ไม่ต้องใช้ visited[] เลย

🤔 แล้วทำไมต้องใช้ Tree?

Linear DS — Array, Linked List, Stack, Queue
เก็บข้อมูลเรียงต่อกันเป็นเส้นตรง — ถ้าอยากค้นหาหรือเข้าถึง ต้องไล่ไปทีละตัว
เมื่อข้อมูลใหญ่ขึ้น time complexity เพิ่มตาม — O(n) สำหรับการค้นหา
↓ ต้องการอะไรที่ดีกว่า
Non-linear DS — Tree
จัดข้อมูลแบบ Hierarchical (ลำดับชั้น) — เข้าถึงได้เร็วกว่าเพราะไม่ต้องไล่ทุกตัว
BST ค้นหาใน O(log n) · Heap ดึง min/max ใน O(1) · Segment Tree query ใน O(log n)
Chapter 02

Terminology & Properties

⏱ ~25 min · ต้องแม่น — โจทย์ CP และ Olympiad ใช้ตลอด

Root
โหนดบนสุด ไม่มี parent — node 1
Parent / Child
1 เป็น parent ของ 2,3,4 / 2 เป็น child ของ 1
Leaf Node
ไม่มีลูก — 3, 5, 6, 7 คือ leaf
Sibling
parent เดียวกัน — 5 กับ 6 เป็น sibling
Depth
จำนวน edge จาก root — depth(5) = 2
Height
edge ยาวสุดลงไปถึง leaf — height(tree) = 2
Subtree
node + ทุก descendant — subtree(2) = {2,5,6}
Degree
จำนวน children — degree(1) = 3, degree(leaf) = 0
Ancestor
ทุก node บนเส้น root→node — ancestor(5) = {1,2}
Descendant
node ที่อยู่ "ใต้" node นั้น — descendant(2) = {5,6}
Path
ลำดับ node ตาม edge — path(1→5) = 1, 2, 5 (ยาว 2 edge)
Internal Node
มีลูกอย่างน้อย 1 ตัว — 1, 2, 4
Level
= depth + 1 (บางตำราเริ่มจาก 1 — ระวังด้วย)
n nodes → n−1 edges
สมบัติหลักของ Tree — 7 nodes = 6 edges เสมอ

📐 Properties สำคัญ

🔗
Number of Edges
Tree ที่มี N nodes จะมี N−1 edges เสมอ — มากกว่านี้คือมี cycle, น้อยกว่านี้คือ disconnected
และมีเส้นทางเดียว (unique path) ระหว่าง node ใดๆ สองตัวเสมอ
📏
Depth of a Node
ความยาวเส้นทาง (นับ edge) จาก root ลงมาถึง node นั้น — root มี depth = 0
📐
Height of the Tree
เส้นทางยาวที่สุดจาก root ลงไปถึง leaf ใดก็ได้ — tree ที่มีแค่ root มี height = 0
🌿
Degree of a Node / Tree
Degree ของ node = จำนวน children — leaf มี degree 0
Degree ของ tree = degree สูงสุดในทุก node
Chapter 03

ประเภทของ Tree

⏱ ~15 min

Tree แบ่งตามจำนวน children ที่แต่ละ node มีได้เป็นหลัก:

Binary Tree
ลูกได้มากสุด 2 ตัว

แต่ละ node มี left และ right child ได้ไม่เกิน 2 ตัว

  • Full BT — ทุก node มีลูก 0 หรือ 2
  • Complete BT — ทุก level เต็มยกเว้น level สุดท้าย (เติมซ้ายก่อน)
  • Perfect BT — ทุก level เต็มหมด, ทุก leaf อยู่ depth เดียวกัน
  • Balanced BT — ต่าง height ซ้าย−ขวา ≤ 1
Ternary Tree
ลูกได้มากสุด 3 ตัว

แต่ละ node มีลูกได้ไม่เกิน 3 ตัว: left, middle, right

  • ใช้น้อยกว่า Binary Tree
  • ตัวอย่าง: Ternary Search Tree
N-ary Tree
ลูกได้ไม่จำกัด

แต่ละ node มีลูกได้ ไม่จำกัดจำนวน — ตรงกับ tree ที่ได้รับจาก input ใน CP

  • เก็บ children ด้วย vector
  • โจทย์ Olympiad ส่วนใหญ่เป็น N-ary

🔑 BST และ Variants ที่เจอใน CP

Typeเงื่อนไขพิเศษใช้ใน
BSTLeft < Root < Right ทุก nodeค้นหา O(log n)
AVL TreeBST + balance อัตโนมัติ (rotate)รับประกัน O(log n)
HeapComplete BT + parent ≤/≥ childrenPriority Queue
Segment TreeArray-based BT เก็บ range aggregateRange Query CP

🌳 Binary Tree — ทำไมสำคัญที่สุด?

Binary Tree เป็น tree ที่ถูกพูดถึงมากที่สุดในเชิงทฤษฎี เพราะ structure เกือบทุกอย่างใน DSA ต่อยอดมาจากมัน

🔍
Binary Search Tree
ค้นหา / insert / delete ใน O(log n) — เป็นพื้นฐานของ set และ map ใน STL
CP: ใช้ set/map แทน BST ตรงๆ เสมอ
⏫
Heap (Priority Queue)
Complete BT ที่ parent ≥ children เสมอ — extract max/min ได้ O(log n)
CP: priority_queue<int> ใช้งานตรงๆ
📊
Segment Tree
Binary Tree เก็บ range aggregate (sum, max, min) — ตอบ range query ได้ O(log n)
CP: สำคัญมาก — Range Sum / Range Max Query
🗜️
Huffman Tree
Full BT ที่ใช้ใน data compression — node ที่ถูกใช้บ่อยอยู่ใกล้ root ที่สุด
CP: ออกเป็นโจทย์ greedy + priority_queue
⚡ สรุป Binary Tree ใน CP

คุณแทบไม่ต้อง เขียน Binary Tree ด้วย struct เอง แต่ต้อง เข้าใจ concept ให้แม่น เพราะ:
Heap → priority_queue · BST → set/map · Segment Tree → array-based implementation · Huffman → greedy + priority_queue

Chapter 04

แนวคิด Tree Traversal

⏱ ~30 min

⚠️ แยกให้ออก — สองสิ่งที่ไม่เหมือนกัน

Pre / In / Postorder = ลำดับการเยี่ยมชมเชิงโครงสร้าง (binary tree เป็นหลัก)
DFS / BFS = algorithm ที่ใช้เดินจริงบน adjacency list ใน CP
→ เกี่ยวข้องกัน แต่คนละบริบท

Binary tree ที่ใช้ในตัวอย่าง — ตัวเลขวงกลมสีคือลำดับที่เยี่ยม:

Preorder — Root → Left → Right

เยี่ยม root ก่อนเสมอ → ลง left subtree → ลง right subtree
ผลลัพธ์: A B D E C F G

⚡ CP Use Case

ใช้ serialize/copy tree — root มาก่อนเสมอ สร้าง tree ใหม่ได้ถ้ามี inorder คู่กัน

Inorder — Left → Root → Right

ลงซ้ายสุดก่อน → root → right — ใน BST ได้ sorted เสมอ!
ผลลัพธ์: D B E A F C G

⚡ CP Use Case

Inorder ของ BST → sorted array ทันที — ใช้ list ค่าทุกตัวเรียงลำดับ หรือ verify ว่าต้นไม้คือ BST จริง

Postorder — Left → Right → Root

ลูกต้องเสร็จก่อน parent เสมอ — root มาสุดท้าย
ผลลัพธ์: D E B F G C A

⚡ CP Use Case

ใช้ตอนคำนวณค่าจาก subtree ขึ้นมา เช่น subtree size, tree DP — ลูกต้องเสร็จก่อนเสมอ

Traversalลำดับผลลัพธ์ (A-G)ใช้ตอน
PreorderRoot → L → RA B D E C F GCopy / Serialize tree
InorderL → Root → RD B E A F C GBST → sorted output
PostorderL → R → RootD E B F G C ASubtree computation / Tree DP
Chapter 05

Representation 3 แบบ

⏱ ~20 min

1. Pointer / Struct

Textbook / ดู Concept

แต่ละ node เก็บ pointer ไปหา children — เห็นโครงสร้างชัด แต่ต้อง manage memory เอง ไม่ใช่ CP style

2. Array Index

Complete BT / Heap

node ที่ index i → left: 2i+1, right: 2i+2 — ใช้ได้เฉพาะ Complete Binary Tree

3. Adjacency List ⭐

CP — ใช้จริง

เก็บ vector ของเพื่อนบ้าน — อ่าน input ได้ตรง เดิน DFS/BFS ได้ทันที ใช้ได้ทั้ง tree และ graph

❌ Struct + Pointer (Textbook)
struct Node {
    int val;
    Node* left;
    Node* right;
};
// ต้อง new/delete เอง
// memory leak ได้ง่าย
// ไม่เหมาะกับ CP
✅ Adjacency List (CP Style)
#include <bits/stdc++.h>
using namespace std;

vector<int> adj[100005];
// node = แค่ตัวเลข 1..n
// ไม่มี new/delete เลย
// อ่าน input ได้ตรงๆ

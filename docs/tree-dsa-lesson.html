Chapter 01

Tree คืออะไร?

⏱ ~20 min

Tree คือโครงสร้างข้อมูลแบบ Hierarchical — ข้อมูลจัดเป็นลำดับชั้นจากบนลงล่าง ต่างจาก Array ที่เรียงตรงๆ หรือ Graph ทั่วไปที่ไม่มีทิศทาง

🔑 มุมมอง Competitive Programming

ใน CP เรามอง Tree ว่าเป็น Graph พิเศษ ที่มีสมบัติครบ 3 ข้อ:

⚡ CP Definition

1. Connected — ทุก node ถึงกันได้
2. Acyclic — ไม่มี cycle
3. n nodes มี n−1 edges เสมอ — มากกว่านี้คือ cycle, น้อยกว่านี้คือ disconnected

โจทย์ CP มักให้ input เป็น n และ n-1 edges แล้วให้เดิน DFS/BFS — ไม่ได้สร้าง struct Node ให้เลย

🌍 ตัวอย่าง Real-world

  • File system — โฟลเดอร์ → subfolders → files
  • HTML DOM — <html> คือ root, <body> คือ child
  • Binary Heap, BST, AVL, Segment Tree — ล้วนเป็น Tree variant ทั้งสิ้น
Chapter 02

Terminology

⏱ ~20 min · ต้องแม่น — โจทย์ CP และ Olympiad ใช้ตลอด

Root
โหนดบนสุด ไม่มี parent — node 1
Parent / Child
1 เป็น parent ของ 2,3,4 / 2 เป็น child ของ 1
Leaf Node
ไม่มีลูก — 3, 5, 6, 7 คือ leaf
Sibling
parent เดียวกัน — 5 กับ 6 เป็น sibling
Depth
จำนวน edge จาก root — depth(5) = 2
Height
edge ยาวสุดลงไปถึง leaf — height(tree) = 2
Subtree
node + ทุก descendant — subtree(2) = {2,5,6}
Degree
จำนวน children — degree(1) = 3, degree(leaf) = 0
Ancestor
ทุก node บนเส้น root→node — ancestor(5) = {1,2}
Internal Node
มีลูกอย่างน้อย 1 ตัว — 1, 2, 4
Level
= depth (บางตำราเริ่มจาก 1 ระวังด้วย)
n nodes
n−1 edges เสมอ → 7 nodes = 6 edges
Chapter 03

แนวคิด Tree Traversal

⏱ ~30 min

⚠️ แยกให้ออก — สองสิ่งที่ไม่เหมือนกัน

Pre / In / Postorder = ลำดับการเยี่ยมชมเชิงโครงสร้าง (binary tree เป็นหลัก)
DFS / BFS = algorithm ที่ใช้เดินจริงบน adjacency list ใน CP
→ เกี่ยวข้องกัน แต่คนละบริบท

Binary tree ที่ใช้ในตัวอย่าง — ตัวเลขวงกลมสีคือลำดับที่เยี่ยม:

Preorder — Root → Left → Right

เยี่ยม root ก่อนเสมอ → ลง left subtree → ลง right subtree
ผลลัพธ์: A B D E C F G

⚡ CP Use Case

ใช้ serialize/copy tree — root มาก่อนเสมอ สร้าง tree ใหม่ได้ถ้ามี inorder คู่กัน

Inorder — Left → Root → Right

ลงซ้ายสุดก่อน → root → right — ใน BST ได้ sorted เสมอ!
ผลลัพธ์: D B E A F C G

⚡ CP Use Case

Inorder ของ BST → sorted array ทันที — ใช้ list ค่าทุกตัวเรียงลำดับ หรือ verify ว่าต้นไม้คือ BST จริง

Postorder — Left → Right → Root

ลูกต้องเสร็จก่อน parent เสมอ — root มาสุดท้าย
ผลลัพธ์: D E B F G C A

⚡ CP Use Case

ใช้ตอนคำนวณค่าจาก subtree ขึ้นมา เช่น subtree size, tree DP — ลูกต้องเสร็จก่อนเสมอ

Traversalลำดับผลลัพธ์ (A-G)ใช้ตอน
PreorderRoot → L → RA B D E C F GCopy / Serialize tree
InorderL → Root → RD B E A F C GBST → sorted output
PostorderL → R → RootD E B F G C ASubtree computation / Tree DP
Chapter 04

Representation 3 แบบ

⏱ ~20 min

1. Pointer / Struct

Textbook / ดู Concept

แต่ละ node เก็บ pointer ไปหา children — เห็นโครงสร้างชัด แต่ต้อง manage memory เอง ไม่ใช่ CP style

2. Array Index

Complete BT / Heap

node ที่ index i → left: 2i+1, right: 2i+2 — ใช้ได้เฉพาะ Complete Binary Tree

3. Adjacency List ⭐

CP — ใช้จริง

เก็บ vector ของเพื่อนบ้าน — อ่าน input ได้ตรง เดิน DFS/BFS ได้ทันที ใช้ได้ทั้ง tree และ graph

❌ Struct + Pointer (Textbook)
struct Node {
    int val;
    Node* left;
    Node* right;
};
// ต้อง new/delete เอง
// memory leak ได้ง่าย
// ไม่เหมาะกับ CP
✅ Adjacency List (CP Style)
#include <bits/stdc++.h>
using namespace std;

vector<int> adj[100005];
// node = แค่ตัวเลข 1..n
// ไม่มี new/delete เลย
// อ่าน input ได้ตรงๆ

Binary Tree Types — ภาพรวม

Typeเงื่อนไขใช้ใน
Full BTทุก node มีลูก 0 หรือ 2 ตัวHuffman Coding
Perfect BTทุก level เต็ม, leaves depth เดียวกันSegment Tree
Complete BTเต็มทุก level ยกเว้นสุดท้าย (เติมซ้ายก่อน)Binary Heap
Balanced BT|height(L) − height(R)| ≤ 1AVL, Red-Black
BSTLeft < Root < Right ทุก nodeค้นหา O(log n)
⚡ CP Note

โจทย์ CP มักให้ tree เป็น adjacency list ทั่วไป — ไม่ต้องรู้ว่า binary/balanced ก็ใช้ DFS/BFS ได้ทุกกรณี

Chapter 05

Basic Operations — พาทำทีละขั้น

⏱ ~25 min · ลองเขียนเองก่อนดู full code

เรียนรู้ 4 operations พื้นฐาน ด้วย struct/pointer ก่อน เพื่อเข้าใจโครงสร้าง — ดู limitation ในบทถัดไป

01
Create — สร้าง Node Structure
นิยาม struct ของ Node เก็บ data และ vector ของ children (N-ary ลูกได้ไม่จำกัด)
C++
struct Node {
    int data;
    vector<Node*> children;

    Node(int x) : data(x) {}
};
02
Insert — เพิ่ม Node เข้า Tree
สร้าง root แล้ว addChild ทีละตัว สร้าง tree จากโค้ด
C++
void addChild(Node* parent, Node* child) {
    parent->children.push_back(child);
}

int main() {
    Node* root = new Node(1);
    addChild(root, new Node(2));
    addChild(root, new Node(3));
    addChild(root->children[0], new Node(4));
}
03
Search — ค้นหา Node
เดิน DFS recursive — ถ้า data ตรงก็ return node นั้น
C++
Node* search(Node* node, int target) {
    if (!node) return nullptr;
    if (node->data == target) return node;

    for (Node* child : node->children) {
        Node* found = search(child, target);
        if (found) return found;
    }
    return nullptr;
}
// O(n) — ต้องเดินทุก node ในกรณีแย่สุด
04
Traverse — เดินผ่านทุก Node
DFS recursive — เปลี่ยนตำแหน่ง cout ได้ preorder/postorder ต่างกัน
C++
void dfs(Node* node) {
    if (!node) return;

    cout << node->data << " ";   // preorder: เยี่ยมก่อนลงลูก

    for (Node* child : node->children)
        dfs(child);

    // ย้าย cout มาตรงนี้ → postorder
}

ลอง implement ทั้ง 4 operations แล้วค่อย verify กับ examples/01-tree/05_basic_operations.cpp

Chapter 06

ข้อจำกัด & เมื่อไรใช้ set / map

⏱ ~15 min

หลังจากเข้าใจ struct/pointer แล้ว ต้องรู้ด้วยว่า เมื่อไรไม่ควรใช้

⚠️ ปัญหาของ struct + pointer ใน CP

Memory Management
ต้อง new/delete ทุกครั้ง — ลืม delete = memory leak
Stack Overflow
Recursive DFS บน tree ลึก 10⁵ nodes — stack เต็มทันที
ไม่ Auto-balance
BST ธรรมดา insert sorted data → Skewed tree → O(n) search
เขียนช้า
ใน contest เวลาจำกัด implement BST เอง 50+ บรรทัดเสียเวลาเปล่า

✅ ใช้ STL แทน — รับประกัน O(log n)

std::set / std::map ใช้ Red-Black Tree ข้างใน — BST ที่ balance ตัวเองอัตโนมัติ

❌ เขียน BST เอง (~60 บรรทัด)
Node* insert(Node* n, int x) { /*...*/ }
Node* search(Node* n, int x) { /*...*/ }
Node* del   (Node* n, int x) { /*...*/ }
// ยังไม่มี balance!
// worst case O(n) ถ้า sorted input
✅ ใช้ std::set (3 บรรทัด)
set<int> s;
s.insert(5);    // O(log n)
s.count(5);     // O(log n) — มีไหม?
s.erase(5);     // O(log n)
// auto-balanced เสมอ, ไม่มี leak

สรุป: เลือกใช้อะไร

สถานการณ์ใช้เหตุผล
Sorted set + insert/delete/searchstd::setO(log n) auto-balanced RB-Tree
Key→value sortedstd::mapO(log n) auto-balanced RB-Tree
Input โจทย์ให้ n nodes + edgesvector<int> adj[]adjacency list — CP standard
อธิบาย concept ให้คนอื่นstruct Nodeเห็นโครงสร้างชัด
⚡ Key Takeaway

BST เองไม่ค่อย implement ใน CP — ใช้ set/map แทน
Tree จาก input โจทย์ — adjacency list เสมอ
struct Node — ใช้อธิบาย concept หรือสอน concept เท่านั้น

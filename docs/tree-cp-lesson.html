ก่อนเริ่ม — มองก่อนเขียน
Tree ใน CP = Undirected Connected Graph ที่มี n−1 edges

ใน CP เราไม่สนใจว่า node ไหนคือ "left child" หรือ "right child" เพราะโจทย์ส่วนใหญ่ให้ tree เป็น N-ary adjacency list ธรรมดา — อ่าน input เหมือน graph ทั่วไปเลย

🌳 View 1 — Rooted Tree
กำหนด root = node 1 — มี parent/child ชัดเจน
ใช้กับ: depth[], parent[], subtree size, Tree DP
pattern: dfs(1, 0)
🕸 View 2 — Unrooted (Graph)
ไม่สน parent — แค่ "เพื่อนบ้าน" ทั้งหมด
ใช้กับ: diameter, path ระหว่าง 2 node
pattern: dfs เหมือน graph + visited[]
📌 Part นี้ใช้ View 1 (Rooted) เป็นหลัก — root = node 1, parent ของ root = 0 (sentinel) เสมอ
Chapter 01

อ่าน Input → Adjacency List

⏱ ~20 min · Pattern พื้นฐานที่สุด — ต้องทำได้ทันที

โจทย์ CP ส่วนใหญ่ให้ input เป็น n ตามด้วย n−1 บรรทัดของ edge:

Standard Input Format
7 1 2 1 3 1 4 2 5 2 6 4 7

Pattern 1 — vector local ใน main

เหมาะกับโจทย์ที่ n ไม่ใหญ่มาก เห็นโครงสร้างชัด:

C++ส่วนสำคัญ
// ประกาศใน main()
vector<vector<int>> adj(n + 1);   // size n+1 เพราะ node นับจาก 1

for (int i = 0; i < n - 1; i++) {
    int u, v;
    cin >> u >> v;
    adj[u].push_back(v);   // undirected → push 2 ทิศทาง
    adj[v].push_back(u);
}

Pattern 2 — Global array (CP จริง)

ใช้เมื่อ n ใหญ่ หรืออยากเข้าถึง adj จาก function อื่นโดยไม่ต้องส่ง parameter:

C++ส่วนสำคัญ
// ประกาศ global (นอก main)
const int MAXN = 1e5 + 5;
vector<int> adj[MAXN];       // init เป็น empty vector อัตโนมัติ

// ใน main() — อ่าน edge เหมือนเดิม
for (int i = 0; i < n - 1; i++) {
    int u, v;
    cin >> u >> v;
    adj[u].push_back(v);
    adj[v].push_back(u);
}
⚡ จุดสำคัญ 2 ข้อ

1. ใช้ adj(n+1) หรือ adj[MAXN] — node นับจาก 1 ถึง n เสมอ
2. Tree undirected → push_back ทั้ง 2 ทิศทางทุกครั้ง

Chapter 02

DFS บน Tree — Skeleton

⏱ ~20 min · เรียน pattern นี้ก่อน — ใช้ใน Ch3 และ Ch4

DFS บน Tree ต่างจาก Graph ตรงที่ ส่ง parent มาด้วย — ไม่ต้องใช้ visited array

💡 ทำไมไม่ต้องใช้ visited?

Tree ไม่มี cycle — ถ้าเราไม่เดินกลับไปหา parent (if v == p: continue) จะไม่มีทางวนซ้ำได้เลย

Skeleton ที่ใช้ซ้ำใน Pattern ทุกอัน

C++ส่วนสำคัญ
void dfs(int u, int p) {
    // ← ทำอะไรกับ u ตรงนี้ (preorder)

    for (int v : adj[u]) {
        if (v == p) continue;   // ไม่เดินกลับหา parent
        dfs(v, u);               // ลงไปหา child
    }

    // ← หรือทำอะไรกับ u ตรงนี้ (postorder)
}

// เรียกครั้งแรก: dfs(1, 0)  ← root=1, parent=0 (sentinel)

Preorder vs Postorder — ต่างกันตรงไหน?

ตำแหน่งชื่อใช้เมื่อ
ก่อน for loopPreorderทำงานกับ node ก่อนลงลูก เช่น print, ส่งค่าลงไป
หลัง for loopPostorderรอลูกเสร็จก่อน เช่น sz[], Tree DP
⚠️ Stack Overflow บน Chain Tree

Recursive DFS จะ stack overflow เฉพาะเมื่อ tree เป็น chain (1→2→3→…→n) depth = n
Balanced tree n=10⁵ มี depth ≈ log₂n ≈ 17 — ปลอดภัยมาก
DFS Iterative เรียนใน Session ถัดไป

Chapter 03

Pattern: depth[] & parent[]

⏱ ~20 min · Pattern ที่ออกบ่อยมากใน Olympiad

ต่อยอด DFS skeleton จาก Ch2 — เพิ่ม global array dep[] และ par[] บันทึกค่าระหว่างเดิน

Global arrays + การส่ง depth ลงไป

C++ส่วนสำคัญ
// global arrays
int dep[MAXN];
int par[MAXN];

void dfs(int u, int p, int d) {
    dep[u] = d;         // บันทึก depth ของ node นี้
    par[u] = p;         // บันทึก parent ของ node นี้

    for (int v : adj[u]) {
        if (v == p) continue;
        dfs(v, u, d + 1);   // ส่ง depth+1 ลงไปหาลูก
    }
}

// เรียก: dfs(1, 0, 0)

หลัง DFS ครั้งเดียว — query ได้ใน O(1) ทุก node:

Output (tree ตัวอย่าง)
node 1 depth=0 parent=0 node 2 depth=1 parent=1 node 3 depth=1 parent=1 node 4 depth=1 parent=1 node 5 depth=2 parent=2 node 6 depth=2 parent=2 node 7 depth=2 parent=4
⚡ ใช้ทำอะไรได้บ้าง

หา depth / parent ของ node ใดก็ได้ใน O(1)
เป็นฐานของ LCA (Lowest Common Ancestor) → ต่อยอดด้วย Binary Lifting
หา height ของ tree = ค่าสูงสุดใน dep[]

Chapter 04

Pattern: Subtree Size

⏱ ~20 min · หัวใจของ Tree DP

นับ node ใน subtree ของทุก node — เป็นพื้นฐานของ Tree DP ทั้งหมด

แนวคิด: sz[u] = 1 + ผลรวม sz[v] ของทุก child v

ต้องใช้ Postorder — ลูกต้องเสร็จก่อน

C++ส่วนสำคัญ
// global array
int sz[MAXN];

void dfs(int u, int p) {
    sz[u] = 1;               // นับตัวเองก่อน

    for (int v : adj[u]) {
        if (v == p) continue;
        dfs(v, u);
        sz[u] += sz[v];        // postorder: รอลูก v เสร็จก่อนบวก
    }
}

// เรียก: dfs(1, 0)
⚡ ทำไมต้อง Postorder

sz[u] += sz[v] อยู่ หลัง dfs(v, u) เสมอ — ต้องรอให้ลูก v คำนวณ sz เสร็จก่อน
นี่คือ pattern หลักของ Tree DP ทุกข้อ: ลูกต้องเสร็จก่อน parent
